# Инструкция для аналитика Botamin

Дашборд ведёт вас от «как дела» к «что тестировать» за 8 шагов.

---

## Шаг 1. Проверьте здоровье проекта

Блок **«Здоровье проекта»** — три карточки вверху страницы:

| Карточка | Что значит | Хорошо когда |
|---|---|---|
| **Квалификация** | % звонков, где бот дошёл до 4-го шага (узнал объём заявок) | растёт ↑ |
| **Технические сбои** | % звонков с техническими причинами завершения (no_answer, network_error и др.) | падает ↓ |
| **Без диалога** | % звонков, где клиент сбросил не успев ничего сказать | падает ↓ |

Каждая карточка показывает **текущее значение** и **стрелку со % отклонения** от среднего по всему периоду данных.

- Зелёная стрелка ↑ у «Квалификации» — хорошо
- Красная стрелка ↑ у «Сбоев» или «Без диалога» — плохо, нужно разбираться

**Тревожный сигнал:** технические сбои > 5% — звоните команде.

---

## Шаг 2. Найдите слабое звено воронки

Блок **«Воронка»** → переключатель **«По шагам»** (активен по умолчанию).

Каждая строка = один шаг воронки:
- **Количество** звонков, дошедших до этого шага
- **% от начала** — сколько из всех звонков добралось до шага
- **% от предыдущего** — конверсия именно этого перехода (главная цифра)

Красный треугольник ▼ и красная полоса — **наихудший переход**, туда смотрите в первую очередь.

**Клик на строку** → открывает Drill-down (детальный разбор этого шага) ниже на странице.

---

## Шаг 3. Убедитесь — проблема системная, не случайность

Кликнули на шаг → откроется **Drill-down**. Перейдите на вкладку **«По дням»**.

График показывает конверсию этого шага по каждому дню периода:
- Пунктирная линия — среднее значение
- **Красные точки** — дни, где конверсия упала ниже 90% от среднего

Если провал на всех днях — системная проблема скрипта, нужно менять подход.
Если провал только 1–2 дня — возможно внешний фактор (выгрузили неудачную базу, праздник).

> Вкладка «По дням» работает только при открытии drill-down по шагу. При открытии по причине — данные недоступны.

---

## Шаг 4. Разберитесь — формат или содержание?

В Drill-down → вкладка **«Причины»** (открывается по умолчанию при клике на шаг).

Два графика рядом:

### График «Активность клиента»
Показывает, сколько реплик успел сказать клиент до того как бросил трубку:

| Картина | Диагноз |
|---|---|
| 0 реплик → 50%+ | Клиент кладёт трубку не вступив в диалог — монолог слишком длинный |
| 1 реплика → много | Клиент попробовал вступить, но бот не зацепил — проблема первых секунд |
| 3+ реплик → 50%+ | Клиент активно участвовал, но ушёл — **проблема содержания**, аргументы не убеждают |

Зелёный цвет = 3+ реплик (хороший сигнал), красный = 0 реплик (плохой).

### График «Причины отвала»
Распределение причин, по которым бот не смог пройти дальше на этом шаге.
- Серая полоса — **«Другое»**: звонки, которые автоматика не смогла классифицировать. Это нормально для пилота — означает «нужно смотреть руками».
- Числа в скобках — абсолютное количество звонков.

---

## Шаг 5. Проверьте на реальных диалогах

Drill-down → вкладка **«Диалоги»**.

### Фильтры
- **Выпадающий список** (слева) — фильтр по причине отвала. Список сформирован из реальных данных текущего шага. «Все причины» — показывает всё.
- **Кнопка «⚠️ Сначала с низкой уверенностью»** — включает сортировку: сначала идут звонки, где автоклассификация была неуверенна (< 70%). Их стоит проверить первыми — возможно, тег неверный.
- Справа — **счётчик** отфильтрованных звонков (показывается до 200 записей).

### Таблица
Столбцы: **Телефон · Дата · Длительность · Причина · Уверенность**.
Значок ⚠️ — уверенность автоклассификации ниже 70%.

### Просмотр диалога
**Клик на строку** → открывается модальное окно с транскриптом:
- Реплики бота (фиолетовые, слева) и клиента (серые, справа)
- Метаданные: дата, длительность, причина и % уверенности
- Плеер аудиозаписи (если запись прикреплена в данных)

Прослушайте 5–10 звонков. Ищите паттерн: **что именно говорит клиент перед тем как уйти?**

---

## Шаг 6. Посмотрите на глобальную картину причин

Воронка → переключатель **«По причинам»**.

Таблица: каждая строка = одна причина, столбцы **Ш1–Ш4** показывают долю этой причины среди всех звонков на данном шаге. Колонка **«Итого»** — суммарное количество звонков с этой причиной по всем шагам.

**Сортировка по «Итого»** — причина с наибольшим числом — ваш главный приоритет.

**Клик на строку** → открывает Drill-down по этой причине (вкладка «Причины» + «Диалоги»).

> Ищите «тихого убийцу»: причина с небольшой долей на каждом шаге, но с большим итоговым числом — значит она присутствует везде понемногу, но в сумме влияет больше всего.

---

## Шаг 7. Сформулируйте и запустите тест

Блок **«Рекомендации»** — автоматически сгенерированные гипотезы на основе данных.

Каждая карточка содержит:
- **Факт** — конкретное наблюдение из данных
- **Гипотеза** — объяснение, почему так происходит
- **Тест** — конкретный A/Б-эксперимент для проверки
- **Ожидаемый эффект** — прогноз улучшения

Карточки **кликабельны** — можно свернуть/развернуть. По умолчанию открыты.

Выберите наиболее убедительную гипотезу и запустите как A/Б-тест.

---

## Шаг 8. Следите за результатом теста

Блок **«A/Б-тесты»** — список активных и завершённых экспериментов.

В строке каждого теста сразу видно:
- Статус: **✓ Значимо** (зелёный) или **⏳ Мало данных** (оранжевый)
- Какой вариант лучше и на сколько %

**Клик на строку → разворачивает детальную таблицу:**

Строки таблицы (сверху вниз):
1. **Звонков** — объём выборки по каждому варианту
2. **Шаг 1 (согласие)** — доля звонков, где клиент согласился говорить
3. **Шаг 1→2, 2→3, 3→4** — конверсия каждого перехода
4. **Квалификация итого** — итоговая конверсия в квалификацию

Стрелки ↑ (зелёные) и ↓ (красные) у варианта Б — разница относительно варианта А.

**Строка статистики** под таблицей:
- **p-value** — вероятность того, что разница случайна. p < 0.05 = статистически значимо (зелёный)
- **95% CI** — диапазон, в котором с вероятностью 95% находится истинный эффект. Если включает ноль — ещё рано делать выводы

---

## Быстрый маршрут (5 минут с утра)

1. **Здоровье** → всё зелёное, нет аномалий?
2. **Воронка → По шагам** → красный треугольник на том же шаге, что вчера?
3. **Клик на проблемный шаг → По дням** → провал системный или разовый?
4. **→ Причины** → 0 реплик растёт или 3+? Что в «Причинах отвала»?
5. **→ Диалоги** → включить «⚠️ Сначала с низкой уверенностью» → прослушать 3–5 звонков
6. **Рекомендации** → появилась новая гипотеза?
